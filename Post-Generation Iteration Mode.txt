🔁 Post-Generation Iteration Mode (Chat-Based UI Evolution)

After the initial screen generation is completed successfully (i.e., SSE stream receives a "complete" event from /generate-angular-app), the UI should transition into an Interactive Evolution Mode.

This mode enables the user to refine, enhance, or modify the generated screen using a conversational chat interface powered by:

POST /aiqod-agent/agent/update-angular-screen

This endpoint follows the same async job model and SSE streaming behavior as the generation endpoint.

---

# 🧠 New UX Flow

Once the first screen is generated:

### Update the existing layout with:

## Two-panel Layout

| Left Panel                     | Right Panel               |

| ------------------------------ | ------------------------- |

| 💬 Chat-based update interface | 🖥 Live Preview (iframe) |

---

# 🧩 LEFT PANEL — Conversational Update Interface

Convert the left prompt panel into a chat-style interface.

This enables the user to evolve the UI using natural language such as:

* "Change theme to dark mode"

* "Add a submit button"

* "Make this mobile friendly" etc.

---

## Chat Behavior

### Conversation Structure

Maintain a message list:

type Message = {

  id: string

  role: "user" | "system"

  content: string

  status?: "pending" | "applied" | "failed"

}

Show:

* Past updates

* Pending update

* Applied updates

This mirrors backend-supported chat_history behavior.

---

## Sending an Update

When the user submits a new chat message:

### 1️⃣ Call Update Endpoint

POST /aiqod-agent/agent/update-angular-screen

Body:

{

  prompt: <user message>,

  project_id: existing_project_id,

  screen_id: existing_screen_id

}

Same identifiers from original generation must be reused.

---

### 2️⃣ Handle Response

Response returns:

{

  success: true,

  job_id: string

}

---

### 3️⃣ Stream Update Progress

Open SSE stream:

GET /aiqod-agent/agent/stream/{job_id}

Just like generation.

Append logs to chat feed in real-time.

---

### 4️⃣ Handle Terminal Events

| Event        | Behavior                                                   |

| ------------ | ---------------------------------------------------------- |

| "log"      | Show inline assistant activity (e.g. "Updating layout...") |

| "retry"    | Show retry indicator                                       |

| "warning"  | Show non-blocking issue                                    |

| "error"    | Mark message as failed                                     |

| "complete" | Update preview iframe                                      |

---

## Chat UI Rules

* Disable input while update job is running

* Show typing / processing indicator

* Show update status badge:

  * Applying...

  * Applied

  * Failed
